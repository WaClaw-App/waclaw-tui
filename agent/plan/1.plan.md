# WaClaw TUI — Parallel Development Plan v1.3

> **Goal:** Build the complete WaClaw TUI (20 screens, 110 states, 17 notifications, 2 global overlays) so that **every pixel matches the `doc/` specifications exactly**. The TUI is the only deliverable — the closed-source backend binary is out of scope; we build the demo backend that drives the TUI through JSON-RPC over stdio.

---

## 0. Guiding Principles

| # | Principle | Why |
|---|-----------|-----|
| P1 | **Doc is law** | If `doc/` says gold flash on jackpot, the code must gold-flash on jackpot. No approximations. |
| P2 | **Charm.sh only** | bubbletea, lipgloss, bubbles, glamour, huh — zero external UI libraries. |
| P3 | **Vertical borderless** | No `│ ┃ ║`, no `─ ━ ─`, no boxes. Hierarchy = brightness + size + motion. |
| P4 | **English codebase, multi-language UI** | All code is English. User-facing display strings support **two locales**: casual Indonesian (default) and casual English. Locale set in `config.yaml`, switchable at runtime. |
| P5 | **Dual protocol** | JSON-RPC 2.0 over stdio (TUI backend) + REST API with Swagger (future web frontend). Shared `pkg/protocol` types generate both transports. |
| P6 | **Interface-first** | Shared types in `pkg/`, break circular deps with interfaces in `engine/`. |
| P7 | **Animation timing is spec** | 300ms screen transition, 4000ms breathing, 120ms stagger — exact values from doc. |
| P8 | **Never red for rejection** | `danger` (#F87171) only for technical problems. Rejection = `text_muted`. |

## 4. Inter-Agent Contracts

### 4.1 Screen Interface

Every screen must implement:

```go
// internal/tui/screen.go
type Screen interface {
    tea.Model                           // Init, Update, View
    ID() protocol.ScreenID              // Returns screen identifier
    SetBus(bus *bus.Bus)                // Receive event bus reference
    HandleNavigate(params map[string]any) error  // Backend navigate command
    HandleUpdate(params map[string]any) error    // Backend update command
}
```

### 4.2 Bus Message Types

```go
// Bus carries these tea.Msg types:
type NavigateMsg struct { Screen protocol.ScreenID; Params map[string]any }
type UpdateMsg   struct { Screen protocol.ScreenID; Params map[string]any }
type NotifyMsg   struct { Type   string; Severity protocol.Severity; Data map[string]any }
type ValidateMsg struct { Errors []string; Warnings []string }
```

### 4.3 Component Dependencies

Screens import from `component/` — they never import from other screen packages. The only shared dependencies are:
- `pkg/protocol/` — shared types
- `internal/tui/style/` — colors, layout, typography
- `internal/tui/component/` — reusable widgets
- `internal/tui/bus/` — event bus

### 4.4 String Constants — Multi-Language

**Code identifiers are always English.** User-facing display strings support **two locales**: casual Indonesian (`id`, default) and casual English (`en`). All strings live in `internal/tui/i18n/` — never inline.

**Locale config** in `~/.waclaw/config.yaml`:
```yaml
locale: id  # "id" (default) or "en"
```

Switchable at runtime via Ctrl+K command palette or `l` key.

**i18n package structure:**
```
internal/tui/i18n/
├── keys.go     # Centralized key constants — compile-time safe, single source of truth
├── i18n.go     # T(key string) string — looks up current locale
├── en.go       # English map: map[string]string
└── id.go       # Indonesian map: map[string]string
```

## 6. Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Animation timing drift | All timing constants in `animation.go` as named consts, not magic numbers |
| Screen state explosion | State machine generated from `doc/22-state-machine.md`, not ad-hoc |
| Component API churn | Phase 2B components frozen before Phase 3 starts |
| Demo backend out of sync | Demo backend uses same `pkg/protocol` types as real backend |
| Visual regression | Per-screen golden test: render View() → compare against expected string |
| Language consistency | English-only codebase. All display strings in `internal/tui/i18n/` with `en` + `id` maps, keyed by constants from `keys.go`, never inline. Locale switchable at runtime. |
| REST/TUI protocol drift | REST API types auto-generated from same `pkg/protocol` types as JSON-RPC. Single source of truth. Swagger spec generated from protocol structs. |
| Type semantic mixing | Domain types (Severity, NotificationType, WorkerPhase, LeadPhase, ConfirmationType) live in `pkg/protocol/types.go` — never reuse `StateID` for non-screen concepts. |
| Magic string proliferation | i18n keys defined as Go constants in `keys.go`. Protocol strings use `protocol.Version`, `protocol.MethodX` constants. No raw strings in transport code. |
